# Bulgunniakhs Detection

Проект разработан в рамках ГИС-хакатона **GIS'IT 2026** как решение кейса №4 **«Выявление фактов деградации многолетней мерзлоты на землях сельскохозяйственного назначения»**.

## 📌 Описание

Программа автоматизирует выявление термокарстовых форм рельефа  на спутниковых снимках высокого разрешения (Канопус-В) на землях сельскохозяйственного назначения. Анализ проводится для двух ключевых участков Республики Саха (Якутия):

- **Амга** (Amga)
- **Юнкор** (Yunkor)

Для каждого участка определены две зоны интереса (AOI):

- **`*_big_AOI`** – обрезка для сельскохозяйственных угодий (территория обработки полей);
- **`*_scene_AOI`** – экстент снимков Канопуса-В, охватывающий исследуемую территорию.

## 🧠 Алгоритм

1. **Предобработка** (`preprocessing.py`)  
   - Чтение всех MS‑снимков Канопус-В.  
   - Построение для каждого снимка 6‑канального композита:  
     `B, G, R, NIR, NDWI, NDVI`.  
   - При наличии DEM – добавление уклон (slope) и экспозиция (aspect) (каналы 7 и 8).  
   - Сохранение композитов в исходной геометрии (без глобального выравнивания).

2. **Классификация** (`classification_model.py`)  
   - Используется **Random Forest** с текстурными признаками (std в окне 3×3 для каждого канала).  
   - Обучение на размеченных полигонах (GeoJSON из папки `scene/`).  
   - Модель сохраняется (`Amga_rf_multisource.pkl`) и может быть переиспользована.  
   - Для каждого композита вычисляется карта вероятностей и бинарная маска (с постобработкой: удаление мелких объектов, морфологическое закрытие).  
   - Маски сохраняются как `*_pred.tif`, карты вероятностей (опционально) – `*_prob.tif`.

3. **Агрегация** (`mask_aggregation.py`)  
   - Все бинарные маски территории приводятся к общему экстенту.  
   - Вычисляется **частота детекции** – доля снимков, на которых пиксель отнесён к объекту.  
   - Сохраняется карта частоты (`degradation_frequency.tif`).

4. **Визуализация и статистика** (`vizualize_results.py`)  
   - Построение графиков: важность признаков, временной тренд площади объектов, ROC‑кривая, матрица ошибок.  
   - Расчёт метрик (PA, UA, F1) по валидационным данным.  
   - Сохранение статистики в CSV (`result/stats/`), графиков – в `result/plots/`.

5. **Полный пайплайн** (`pipeline.py`)  
   - Последовательный запуск всех этапов с возможностью пропустить шаги и переобучить модель.

## ⚙️ Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/your-username/bulgunniakhs-detection.git
cd bulgunniakhs-detection
```   
## 2. Скачивание исходных данных

### 2.1. Спутниковые снимки и кадастр

Исходные снимки Канопус-В, кадастровые CSV и AOI‑полигоны предоставлены организаторами кейса. Скачайте архив по ссылке:

🔗 **Яндекс Диск**: [https://disk.yandex.ru/d/uTWkCmpbwfi3wg](https://disk.yandex.ru/d/uTWkCmpbwfi3wg)

Распакуйте содержимое в корень проекта, чтобы появились папки:
- `GISIT_Якутск_Данные/` – спутниковые снимки;
- `Земельный Кадастр_Якутия/` – кадастровые CSV;
- `scene/` – AOI‑полигоны (уже есть в репозитории, но можно обновить).

### 2.2. Цифровая модель рельефа (ArcticDEM)

Для добавления морфометрических признаков (уклон, экспозиция) рекомендуется использовать DEM высокого разрешения. В проекте применяются тайлы **ArcticDEM** (2 м разрешение), покрывающие участки Амга и Юнкор:

- `72_49_2_1_2m_v4.1` (территория Амга)
- `73_42_1_2_2m_v4.1` (территория Юнкор)

**Скачивание и подготовка DEM:**

1. Загрузите нужные тайлы с сайта [ArcticDEM](https://www.pgc.umn.edu/data/arcticdem/) или из другого источника.
2. Распакуйте тайлы (обычно в формате GeoTIFF).
3. При необходимости объедините тайлы в один файл для каждой территории (например, с помощью `gdal_merge.py` или QGIS).
4. Переименуйте полученные файлы:
   - для Амги: `Amga_dem.tif`
   - для Юнкора: `Yunkor_dem.tif`
5. Поместите файлы в корень проекта.

> **Примечание:** Если DEM отсутствует, скрипт `preprocessing.py` создаст композиты только из спектральных каналов (6 каналов). Для улучшения качества классификации настоятельно рекомендуется добавить DEM.

## 3. Установка зависимостей

Создайте виртуальное окружение (рекомендуется) и установите библиотеки:

```bash
pip install -r requirements.txt
```
К основным зависимостям относятся:
*  numpy, rasterio, geopandas, shapely, scikit-learn, scikit-image, scipy, matplotlib, seaborn, tqdm, joblib, pandas, albumentations, torch, segmentation_models_pytorch

## 4. Подготовка данных

- Убедитесь, что папки с данными на месте (см. шаг 2.1).
- Если вы скачали DEM, положите файлы `Amga_dem.tif` и/или `Yunkor_dem.tif` в корень проекта.
- (Опционально) Для обучения модели используйте размеченные полигоны в `scene/27082019.geojson`, `scene/06052025.geojson` (Амга) и `scene/13072021.geojson` (Юнкор). При необходимости добавьте свои.

## 5. Запуск полного пайплайна

```bash
python pipeline.py
```
**Доступные аргументы:**

| Аргумент | Действие |
|----------|----------|
| `--skip-preprocessing` | пропустить создание композитов |
| `--skip-classification` | пропустить классификацию (если маски уже есть) |
| `--skip-aggregation` | пропустить агрегацию |
| `--skip-visualization` | пропустить визуализацию |
| `--skip-cadastre` | пропустить обработку кадастровых CSV |
| `--train-model` | переобучить модель Random Forest (первый запуск обязателен) |

Пример:

```bash
# Переобучить модель и выполнить классификацию (композиты уже созданы)
python pipeline.py --skip-preprocessing --train-model
```

## 6. Альтернативный запуск (по шагам)

- **Только композиты:** `python preprocessing.py`
- **Только классификация (с загрузкой существующей модели):** `python classification_model.py`
- **Переобучение модели:** добавьте в `classification_model.py` флаг `TRAIN_NEW_MODEL = True` или запустите `pipeline.py --train-model`.
- **Только агрегация:** `python mask_aggregation.py`
- **Только визуализация:** `python vizualize_results.py`

## 7. Результаты

- **Композиты** – многоканальные GeoTIFF для каждого снимка (6 или 8 каналов).  
- **Бинарные маски** (`*_pred.tif`) – выделение объектов деградации.  
- **Карты вероятностей** (`*_prob.tif`) – степень уверенности модели.  
- **Карта частоты** (`degradation_frequency.tif`) – доля снимков, где пиксель отнесён к объекту.  
- **CSV-статистика** – площади объектов по датам (га), метрики модели.  
- **Графики** – важность признаков, временной тренд, ROC-кривая, матрица ошибок.  
- **Векторные слои** – кадастровые объекты, обрезанные по AOI (`amga_clipped.geojson`, `yunkor_clipped.geojson`).
